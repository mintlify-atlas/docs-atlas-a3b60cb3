## Overview

Company logos are stored as SVG components in `src/components/company-logos.tsx`. This guide shows you how to add a new logo with proper accessibility and styling.

## Logo Requirements

<Check>**SVG format** - Use SVG for scalability and styling flexibility</Check>
<Check>**Accessible** - Include proper title and ARIA attributes</Check>
<Check>**Responsive** - Set appropriate width and height</Check>
<Check>**Theme-aware** - Support both light and dark modes</Check>

## Adding a Logo

<Steps>
  <Step title="Open the company-logos.tsx file">
    Navigate to `src/components/company-logos.tsx`
    
    ```bash
    code src/components/company-logos.tsx
    ```
  </Step>

  <Step title="Add your logo to the companyLogos object">
    Add a new entry with your company's `logoType` as the key:
    
    ```typescript
    export const companyLogos: Record<string, React.ReactNode> = {
      // ... existing logos
      yourcompany: (
        <svg
          aria-labelledby="yourcompany-logo-title"
          className="text-zinc-900 dark:text-zinc-100"
          height="30"
          viewBox="0 0 100 100"
          width="30"
          xmlns="http://www.w3.org/2000/svg"
        >
          <title id="yourcompany-logo-title">YourCompany</title>
          <path fill="currentColor" d="..." />
        </svg>
      ),
    };
    ```
  </Step>

  <Step title="Test your logo">
    Run the development server and navigate to your company page:
    
    ```bash
    pnpm dev
    ```
    
    Visit `http://localhost:3000/yourcompany` to see your logo.
  </Step>
</Steps>

## Logo Template

Use this template when adding a new logo:

```typescript src/components/company-logos.tsx
export const companyLogos: Record<string, React.ReactNode> = {
  yourcompany: (
    <svg
      aria-labelledby="yourcompany-logo-title"
      className="text-zinc-900 dark:text-zinc-100"
      height="30"
      viewBox="0 0 [width] [height]"
      width="30"
      xmlns="http://www.w3.org/2000/svg"
    >
      <title id="yourcompany-logo-title">YourCompany</title>
      {/* Your SVG paths here */}
      <path fill="currentColor" d="..." />
    </svg>
  ),
};
```

### Template Attributes

<ResponseField name="aria-labelledby" type="string" required>
  Links the SVG to its title for screen readers
  
  - Format: `"yourcompany-logo-title"`
  - Must match the `id` in the `<title>` element
</ResponseField>

<ResponseField name="className" type="string" required>
  Tailwind classes for theming
  
  - For logos that need theme support: `"text-zinc-900 dark:text-zinc-100"`
  - For colored logos: Use specific color classes or omit
</ResponseField>

<ResponseField name="height" type="string" required>
  Logo height in pixels
  
  - Standard: `"30"`
  - Adjust if needed for visual balance
</ResponseField>

<ResponseField name="width" type="string" required>
  Logo width in pixels
  
  - Standard: `"30"`
  - Can be different from height for aspect ratio
</ResponseField>

<ResponseField name="viewBox" type="string" required>
  SVG coordinate system
  
  - Format: `"0 0 [width] [height]"`
  - Use the original SVG's viewBox values
</ResponseField>

<ResponseField name="xmlns" type="string" required>
  XML namespace for SVG
  
  - Always: `"http://www.w3.org/2000/svg"`
</ResponseField>

## Theme Support

Most logos should support both light and dark themes. There are several approaches:

### Approach 1: currentColor (Recommended)

Use `currentColor` with theme-aware text colors:

```typescript
<svg className="text-zinc-900 dark:text-zinc-100">
  <path fill="currentColor" d="..." />
</svg>
```

**Pros**: Simple, consistent, works with Tailwind's color system

**Cons**: All paths have the same color

### Approach 2: Class-based Colors

Use Tailwind classes directly on paths:

```typescript
<svg>
  <path className="fill-zinc-900 dark:fill-zinc-100" d="..." />
  <path className="fill-blue-500 dark:fill-blue-400" d="..." />
</svg>
```

**Pros**: Different colors for different paths, full control

**Cons**: More verbose, harder to maintain

### Approach 3: Opacity Variations

Use opacity with currentColor:

```typescript
<svg className="text-zinc-900 dark:text-zinc-100">
  <path fill="currentColor" opacity="0.6" d="..." />
  <path fill="currentColor" d="..." />
</svg>
```

**Pros**: Creates depth while maintaining theme support

**Cons**: Limited to opacity variations

### Approach 4: Specific Colors

Use specific colors that work in both modes:

```typescript
<svg>
  <path fill="#00B4D8" d="..." />
  <path fill="#0077B6" d="..." />
</svg>
```

**Pros**: Brand-accurate colors

**Cons**: May not look good in all themes

<Warning>
Test your logo in both light and dark modes to ensure good contrast and visibility.
</Warning>

## Sizing Guidelines

### Standard Size

Most logos use a standard 30x30 size:

```typescript
<svg height="30" width="30" viewBox="0 0 100 100">
```

### Wider Logos

For horizontal logos, increase the width:

```typescript
<svg height="30" width="60" viewBox="0 0 200 100">
```

### Taller Logos

For vertical logos, increase the height:

```typescript
<svg height="40" width="30" viewBox="0 0 100 150">
```

<Tip>
Maintain the original aspect ratio by adjusting both width/height and viewBox proportionally.
</Tip>

## Accessibility

### Title Element (Required)

Every logo must have a `<title>` element:

```typescript
<svg aria-labelledby="yourcompany-logo-title">
  <title id="yourcompany-logo-title">YourCompany</title>
  {/* paths */}
</svg>
```

The title:
- Provides a text description for screen readers
- Should be the company name
- Must have an `id` that matches the `aria-labelledby` attribute

### ARIA Attributes

<ResponseField name="aria-labelledby" type="string" required>
  Links the SVG to its title element
  
  ```typescript
  aria-labelledby="yourcompany-logo-title"
  ```
</ResponseField>

<ResponseField name="aria-hidden" type="boolean" optional>
  Hides decorative logos from screen readers (use sparingly)
  
  ```typescript
  aria-hidden="true"
  ```
  
  Only use if the logo is purely decorative and the company name is shown in text nearby.
</ResponseField>

<ResponseField name="role" type="string" optional>
  Specifies the element role
  
  ```typescript
  role="img"
  ```
  
  Usually not needed as `<svg>` has implicit role of `img`.
</ResponseField>

## Optimizing SVGs

Before adding your logo, optimize it:

<Steps>
  <Step title="Remove unnecessary attributes">
    Remove:
    - `id` attributes (except on title)
    - `class` attributes from original SVG
    - `style` attributes
    - XML comments
    - Unnecessary `<g>` wrappers
  </Step>

  <Step title="Simplify paths">
    Use tools like [SVGOMG](https://jakearchibald.github.io/svgomg/) to:
    - Merge paths where possible
    - Remove redundant points
    - Round numbers to reduce file size
  </Step>

  <Step title="Use currentColor">
    Replace hardcoded colors with `currentColor`:
    
    ```typescript
    // Before
    fill="#000000"
    
    // After
    fill="currentColor"
    ```
  </Step>

  <Step title="Test the result">
    Make sure the optimized logo still looks correct.
  </Step>
</Steps>

## Common Issues

<AccordionGroup>
  <Accordion title="Logo is too small/large">
    **Issue**: Logo doesn't match the size of other logos
    
    **Fix**: Adjust the `width` and `height` attributes:
    
    ```typescript
    // Standard size
    <svg height="30" width="30">
    
    // Wider logo
    <svg height="30" width="60">
    
    // Taller logo
    <svg height="40" width="30">
    ```
  </Accordion>

  <Accordion title="Logo is clipped or has extra space">
    **Issue**: The `viewBox` doesn't match the logo's actual bounds
    
    **Fix**: Use the original SVG's `viewBox` values or calculate the correct bounds:
    
    ```typescript
    // The viewBox defines the coordinate system
    // Format: "minX minY width height"
    viewBox="0 0 100 100"
    ```
  </Accordion>

  <Accordion title="Logo doesn't support dark mode">
    **Issue**: Logo uses hardcoded colors that don't work in dark mode
    
    **Fix**: Use one of the theme support approaches:
    
    ```typescript
    // Option 1: currentColor with theme classes
    <svg className="text-zinc-900 dark:text-zinc-100">
      <path fill="currentColor" />
    </svg>
    
    // Option 2: Class-based colors
    <svg>
      <path className="fill-zinc-900 dark:fill-zinc-100" />
    </svg>
    ```
  </Accordion>

  <Accordion title="Logo is not accessible">
    **Issue**: Screen readers can't identify the logo
    
    **Fix**: Add proper title and ARIA attributes:
    
    ```typescript
    <svg aria-labelledby="yourcompany-logo-title">
      <title id="yourcompany-logo-title">YourCompany</title>
      {/* paths */}
    </svg>
    ```
  </Accordion>

  <Accordion title="Logo type doesn't match">
    **Issue**: The key in `companyLogos` doesn't match the `logoType` in your JSON
    
    **Fix**: Make sure they match exactly:
    
    ```json company.json
    {
      "logoType": "yourcompany"
    }
    ```
    
    ```typescript company-logos.tsx
    export const companyLogos: Record<string, React.ReactNode> = {
      yourcompany: (/* ... */),
    };
    ```
  </Accordion>
</AccordionGroup>
